COMPONENT HIERARCHY:

App (QueryClientProvider)
└── AppContent
    ├── CaseList View (default)
    │   ├── CaseFilters
    │   └── CaseList
    │       └── StatusBadge (per row)
    │
    └── CaseDetail View (when case selected)
        ├── StatusBadge
        ├── IssuesList
        ├── OrderPreview
        └── AuditTimeline

HOOKS USAGE:

- useTeamsAuth: App initialization
- useRole: AppContent (role-based rendering)
- useCases: AppContent (list view)
- useCase: CaseDetail (detail view)

API FLOW:

1. Teams SDK → getAuthToken()
2. authService → exchange Teams token → API token
3. apiClient → authenticated API requests
4. React Query → cache & state management

AUTHENTICATION:

Teams (Tenant B) → SSO Token
    ↓
API (Tenant A) → Exchange Token
    ↓
Backend APIs → Authenticated Requests

ROLE-BASED VIEWS:

SalesUser:
- My Cases only
- No team filters
- Can create drafts for own cases

SalesManager:
- All team cases
- Salesperson filter
- Can create drafts for any case

OpsAuditor:
- All cases (read-only)
- Cannot create drafts
- Can download audit bundles
